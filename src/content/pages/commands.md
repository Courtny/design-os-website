# Commands

Type `/` in Claude Code to see all commands. In Cursor, reference skills with `@.claude/skills/<name>/SKILL.md`.

## Core Design Workflows

| Command | What it does |
|---------|-------------|
| `/connect-mcps` | Connect Figma MCP and Storybook for real-time context |
| `/ux-brief` | Create a design brief from a problem statement or PRD |
| `/critique-prep` | Build a critique agenda with feedback prompts |
| `/design-spec-draft` | Draft a UX spec with flows, states, and edge cases |
| `/prototype` | Generate HTML/CSS/JS or React prototypes using design system components |
| `/research-synthesis` | Turn usability sessions or research into structured insights |
| `/accessibility-review` | WCAG 2.1 AA and heuristic review tied to your principles |
| `/meeting-notes` | Structure critique and design sync notes into action items |

## Design Critique

| Command | What it does |
|---------|-------------|
| `/visual-critique` | Critique a design from an attached image. No MCP required. |
| `/figma-critique` | Critique a Figma file or frame via Figma MCP. Optional comment posting. |

## Quality and Review

| Command | What it does |
|---------|-------------|
| `/color-review` | Semantic color, token adherence, and accessibility review |
| `/content-review` | Microcopy, labels, errors, and content hierarchy vs. writing style files |
| `/handoff-check` | Pre-handoff completeness check for engineering |
| `/tufte-viz` | Ideate or critique charts and dashboards using Tufte principles |
